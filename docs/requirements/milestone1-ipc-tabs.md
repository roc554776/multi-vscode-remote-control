# Milestone 1: タブ操作機能 要件定義

## 機能要件

### REQ-1: 外部通信機能

- REQ-1.1: 外部プログラムから VSCode 拡張機能と通信できること
- REQ-1.2: 拡張機能は VSCode 起動時に自動的に通信可能な状態になること
- REQ-1.3: 複数の外部プログラムから同時接続可能であること（nice to have）

### REQ-2: タブ情報取得機能

- REQ-2.1: 現在開いているすべてのタブの一覧を取得できること
- REQ-2.2: 各タブについて以下の情報を取得できること:
  - ファイルパス（URI）
  - タブのラベル（表示名）
  - アクティブかどうか
  - ダーティ（未保存変更あり）かどうか
- REQ-2.3: タブグループごとに情報を取得できること（nice to have）

### REQ-3: タブ操作機能

- REQ-3.1: 指定したタブを閉じることができること
- REQ-3.2: タブを閉じる際、未保存の変更がある場合は保存するか破棄するか選択できること
- REQ-3.3: 複数のタブを一括で閉じることができること（nice to have）

### REQ-4: 動作確認

- REQ-4.1: 拡張機能が正常に起動しているか確認できること
- REQ-4.2: 動作確認手順がドキュメント化されていること
- REQ-4.3: 動作確認を自動化するスクリプトが存在すること

## 非機能要件

### NFR-1: 信頼性

- NFR-1.1: 通信エラー時に適切なエラーメッセージを返すこと
- NFR-1.2: 拡張機能のクラッシュが VSCode 本体に影響しないこと

### NFR-2: 開発体験

- NFR-2.1: 拡張機能のインストール手順が明確であること
- NFR-2.2: 開発中のリロードが容易であること

## 受け入れ基準

1. 外部プログラムから拡張機能に接続し、タブ一覧を取得できる
2. 取得した情報に基づいて特定のタブを閉じることができる
3. 上記の動作確認を手順書またはスクリプトで再現できる
